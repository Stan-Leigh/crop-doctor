[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through how it works under the hood. 
(pause)
First, the user submits their plant query, which goes straight into the security checkpoint node where we automatically scrub any sensitive personal information and verify that no malicious jailbreaks or restricted substances are being requested.
Once cleared, the clean query enters the diagnose node, running our main orchestrator agent which delegates the query to the specialized diagnostician sub-agent using an ADK agent tool.
The diagnostician agent uses our Model Context Protocol server to fetch real-time soil metrics and weather forecasts to contextualize the diagnosis.
Next, instead of giving a recommendation blindly, the workflow pauses at the expert review node for a Human-in-the-Loop approval step, ensuring a real human extension worker approves the diagnosis.
If approved, the workflow routes to the treatment node where the orchestrator runs the specialized treatment specialist sub-agent.
Finally, the treatment specialist queries our local MCP treatment database and outputs the final response with safe, organic remedies.
